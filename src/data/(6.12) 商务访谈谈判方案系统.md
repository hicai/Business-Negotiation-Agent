# PRD – 商务访谈/谈判方案智能体 v1.0

## 1. 背景与目标

- **背景**：
  - 商务人员（尤其是新入职员工）在见客户前缺乏系统的准备框架，容易遗漏关键信息
  - 现有系统已有客户画像录入功能，但缺乏AI驱动的深度分析和谈判辅助
  - 运营团队积累的问答清单需要有效沉淀并转化为可复用的谈判指导

- **业务目标**：
  - 提升商务人员见客户前的准备效率与质量
  - 标准化谈判流程，降低因个人经验差异导致的谈判效果波动
  - 积累并复用商务谈判知识资产

- **成功指标**：
  - 商务人员准备时间缩短≥50%（目标：10分钟内完成客户方案准备）
  - 商务谈判成功率提升≥15%（通过话术指导标准化）
  - 运营负责人Q&A清单复用率≥80%

## 2. 范围

- **包含**：
  - 客户基本资料输入模块（复用现有客户画像入口）
  - AI分析引擎：品牌商业画布生成、痛点挖掘、同类品牌参考方案匹配
  - 产品/服务智能匹配模块
  - 谈判话术指导生成模块
  - 运营后台：Q&A清单管理与维护

- **不包含**：
  - 客户关系管理（CRM）深度集成
  - 实际谈判过程的实时辅助（如实时语音转写提示）
  - 财务报表或法务条款生成

## 5. 功能需求

### 5.1 客户资料输入

- **流程描述**：
  1. 商务人员选择或新建客户档案
  2. 系统展示标准信息框架（基于行业知识库）
  3. 商务人员补充公开信息之外的关键客户画像
  4. 提交后触发AI分析

- **业务规则**：
  - 必填字段：客户名称、行业、规模、近期动态
  - 选填字段：既往合作历史、竞争对手信息、内部决策链
  - 同一客户30天内不重复生成（避免资源浪费）

- **输入/前置**：客户基础资料（名称/行业/规模/近期动态）
- **输出/后置**：触发AI分析任务，返回分析报告

---

### 5.2 AI分析引擎

#### 5.2.1 品牌商业画布生成

- **流程描述**：
  1. AI接收客户资料
  2. 调用行业知识库进行结构化分析
  3. 生成品牌商业画布（9大模块）

- **业务规则**：
  - 输出标准格式：价值主张、客户细分、渠道、客户关系、收入流、核心资源、关键活动、关键合作、成本结构
  - 置信度低于60%的字段需标注"待核实"
  - 分析时效：<30秒

- **输入**：客户基础资料、行业知识库
- **输出**：《品牌商业画布》文档

#### 5.2.2 痛点信息挖掘

- **流程描述**：
  1. AI分析客户近期动态、行业趋势、竞争对手情况
  2. 识别潜在痛点并归类

- **业务规则**：
  - 痛点分类：战略层（业务方向）、运营层（执行效率）、技术层（系统能力）、财务层（成本收益）
  - 每个痛点需标注优先级（P0/P1/P2）
  - 至少输出3个痛点，最多不超过8个

- **输入**：客户动态、行业数据、竞争对手信息
- **输出**：《痛点分析报告》

#### 5.2.3 同类品牌参考方案

- **流程描述**：
  1. 系统匹配同类行业、同等规模的其他品牌案例
  2. 提取可复用的运营策略

- **业务规则**：
  - 案例库来源：运营负责人录入的Q&A清单、公开行业案例
  - 每个参考方案需标注来源
  - 匹配度低于50%的案例不展示

- **输入**：客户画像、案例库
- **输出**：《参考运营方案》列表（最多5个）

---

### 5.3 产品/服务智能匹配

- **流程描述**：
  1. 基于客户痛点和需求，匹配自身产品/服务
  2. 生成推荐方案列表及优先级

- **业务规则**：
  - 匹配逻辑：痛点 → 产品功能 → 客户价值
  - 每个推荐需附带价值说明
  - 最多推荐5个产品/服务

- **输入**：痛点分析结果、产品目录
- **输出**：《产品推荐列表》

---

### 5.4 谈判话术指导生成

- **流程描述**：
  1. AI综合客户画像、痛点、推荐产品生成话术指导
  2. 基于Q&A清单生成可能的问题应答

- **业务规则**：
  - 话术结构：开场白 → 痛点切入 → 方案呈现 → 异议处理 → 促成
  - Q&A覆盖常见问题≥10个场景
  - 支持话术下载为本地文件（PDF/Word）

- **输入**：客户画像、痛点分析、产品推荐、Q&A清单
- **输出**：《谈判话术指导》文档

---

### 5.5 运营后台 - Q&A清单管理

- **流程描述**：
  1. 运营负责人登录后台
  2. 新增/编辑/删除Q&A条目
  3. 设置Q&A与痛点类型的关联

- **业务规则**：
  - Q&A状态：草稿、待审核、已发布
  - 已发布内容修改需记录版本
  - 支持批量导入（Excel格式）

- **输入**：Q&A原始内容
- **输出**：结构化的Q&A清单库

## 1. 背景与目标

### 背景

商务人员在客户拜访前需要花费大量时间准备，包括分析客户情况、了解行业动态、制定谈判策略等。尤其对于新入职的商务人员，缺乏经验可能导致临场遗忘关键步骤，影响谈判效果。

当前系统已实现客户画像录入功能，并已扩展AI分析能力，可通过企查查、官网、公众号、小红书等多渠道获取客户信息，生成包含企业基本信息、企业规模、会员现状、竞品合作、营销活动等维度的深度分析报告。

### 业务目标

- 构建一个AI驱动的商务访谈/谈判方案智能体系统
- 帮助商务人员快速准备客户拜访材料
- 提升谈判成功率，降低因个人经验差异导致的效果波动

### 成功指标

| 指标 | 目标 |
|------|------|
| 商务人员准备时间缩短 | ≥50% |
| 新入职商务人员谈判成功率提升 | ≥30% |
| 系统使用率 | ≥80% |

### 包含

- **企业基本信息分析**（品牌定位、市场份额、发展阶段、区域布局、竞争优势）
- **企业规模分析**（注册资本、员工人数、年收入、主营业务）
- **会员现状分析**（会员体系、规模、活跃率、权益）
- **竞品合作分析**（合作情况、合作伙伴）
- **营销活动分析**（历史活动、效果评估）

## 5. 功能需求

### 5.1 客户资料录入

**流程描述：**

1. 商务人员进入系统，点击"新建方案"
2. 填写客户基本信息（公司名称、行业、规模、联系人等）
3. 系统保存并展示录入的客户信息
4. 触发AI多维度分析

**业务规则：**

- 公司名称为必填项
- 行业需从预设列表中选择
- 支持批量导入客户资料

**输入/前置：**

- 客户公司名称
- 行业分类
- 公司规模
- 联系人信息
- 拜访目的

**输出/后置：**

- 客户资料记录保存成功
- 进入方案生成页面

---

### 5.2 多渠道外部数据获取

**流程描述：**

1. 系统根据客户公司名称自动触发多渠道联网搜索
2. 从企查查获取企业工商信息（注册资本、员工规模等）
3. 从官网获取主营业务和组织架构信息
4. 从公众号获取营销动态和活动信息
5. 从小红书获取用户评价和市场口碑
6. 整合外部信息到客户分析中

**业务规则：**

- 联网获取为自动触发
- 各渠道数据独立获取，任一渠道失败不影响其他渠道
- 外部数据需标注来源
- 数据获取状态可视化展示（已获取/未收录）

**输入/前置：**

- 客户公司名称
- 网络连接正常

**输出/后置：**

- 多渠道外部信息摘要
- 各渠道数据获取状态

---

### 5.3 企业基本信息分析

**流程描述：**

1. 系统根据录入的客户信息，结合多渠道获取的数据
2. AI分析客户品牌的基本属性、市场定位、发展阶段等
3. 生成结构化的企业画像报告

**业务规则：**

- 整合企查查、官网等多渠道数据
- 分析结果需包含：品牌定位、市场份额、发展阶段、区域布局、竞争优势
- 置信度较低的字段需标注"待核实"

**输入/前置：**

- 客户基本资料
- 多渠道外部数据

**输出/后置：**

- 企业基本信息分析报告（长文案格式）

---

### 5.4 企业规模分析

**流程描述：**

1. 基于企查查等渠道获取的数据
2. 分析企业的注册资本、员工规模、年收入等
3. 梳理企业的主营业务领域

**业务规则：**

- 数据来源主要为企查查
- 主营业务需分条列示

**输入/前置：**

- 企查查企业信息
- 公开财务数据

**输出/后置：**

- 企业规模分析报告（长文案格式）

### 5.5 会员现状分析

**流程描述：**

1. 从官网、公众号、小红书等渠道获取会员体系信息
2. 分析会员规模、活跃率、权益类型等
3. 评估会员体系成熟度

**业务规则：**

- 若无会员体系，需标注"尚未建立"
- 会员权益需分条列示
- 活跃率数据可能无法获取，需标注"暂无数据"

**输入/前置：**

- 官网会员页面信息
- 公众号活动数据
- 用户评价

**输出/后置：**

- 会员现状分析报告（长文案格式）

**业务规则：**

- 合作信息需标注来源

- 竞品合作分析报告（长文案格式）

**流程描述：**

3. 评估市场推广能力

**业务规则：**

- 活动时间范围：近6个月

**输入/前置：**

- 公众号历史推文

**输出/后置：**

- 营销活动分析报告（长文案格式）

onMounted(() => {

### 5.1 客户资料录入（精简版）

**流程描述**：
1. 商务人员进入系统，点击"新建方案"

- 客户品牌名称
- 所属行业分类
- 当前业务进度

- 生成初步方案
- 自动跳转至AI对话页面

### 5.2 AI方案生成

**流程描述**：
1. 系统根据录入的客户信息，自动检索品牌行业知识库
2. AI分析客户品牌的基本属性、市场定位等信息
3. 生成精简的品牌画像报告和痛点分析
4. 匹配对应阶段的谈判话术

**业务规则**：
- 信息输出需精简，避免信息过载

**输入/前置**：
- 客户基本资料
- 品牌行业知识库数据

**输出/后置**：
- 精简版客户品牌画像报告
- 痛点分析摘要
- 匹配的谈判话术

### 5.3 AI对话交互

**流程描述**：
1. 方案生成后自动跳转至AI对话页面
2. 显示AI思考过程可视化
3. 用户可通过对话框与AI进行交互
4. 当AI检测到信息不足时，主动向用户提问
5. 支持点击方案模块进行单独修改

**业务规则**：
- AI提问次数上限为5次
- 对于不确定或缺失的信息，必须通过提问获取，严禁编造

- AI思考过程需清晰展示给用户

**输入/前置**：
- 初步生成的方案数据
- 用户输入的补充信息

**输出/后置**：
- AI对话回复
- 更新后的方案内容
- 对话历史记录

1. 在AI对话页面左侧展示历史对话记录

**业务规则**：
- 历史记录需完整保存，支持上下滚动查看

**输入/前置**：
- 当前对话内容

**输出/后置**：
- 完整的历史对话记录列表

### 5.5 谈判话术指导

1. 根据客户情况和业务进度，生成针对性的谈判话术

- 根据选择的业务进度动态展示对应阶段话术
- 支持查看完整的FAQ和反问话术列表

**输入/前置**：
- 客户品牌画像
- 当前业务进度
- 谈判目标

**输出/后置**：
- 当前阶段话术
- 客户常问问题及标准应答
- 我方反问话术
- 通用谈判流程话术

### 5.2 AI方案生成（含品牌画像标签）

5. 匹配对应阶段的谈判话术

**业务规则**：
- 信息输出需精简，避免信息过载
- 优先从内部知识库获取数据
- 品牌画像标签需包含至少5个维度
- 分析结果包含关键维度即可，无需过度展开

**输入/前置**：
- 客户基本资料

- 痛点分析摘要
- 匹配的谈判话术

**流程描述**：
1. AI基于输入的品牌名称和行业分类

4. 生成标准化的品牌画像标签

**业务规则**：
- 标签需包含多个维度（基础属性、生态关系、运营协同等）

**输入/前置**：
- 品牌名称
- 行业分类

**输出/后置**：
- 品牌画像标签列表（含维度、标签、说明）

**实现机制：**
- `analysis` 是 `ref` 响应式变量
- 修改 `analysis.value.xxx` 自动触发Vue的响应式系统
- 左侧模块组件监听数据变化并重新渲染
